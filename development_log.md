# Development Log


## v1.0.0 (2026-03-24)

첫 공식 릴리즈. 주요 내용은 다음과 같습니다.

### 기능 구현

- 작업 의뢰 기본 흐름 구현: 의뢰 등록 → 팀장 승인 → 작업자 수락/반려 → 작업 진행/완료까지 전체 프로세스를 구현하고, 각 단계 전환 시 관련자에게 메일 알림을 보내도록 하였습니다.
- 카드형 대시보드: 시작전/진행중/완료 세 컬럼으로 작업 현황을 구분하고, 기간·검색·완료/반려 토글 등 필터 기능을 추가했습니다. 카드에는 상태 배지, 제목, 요청자/팀, 요청일/희망완료일이 표시되며 임박·지연 여부를 색상으로 강조합니다.
- 검색 기능 개선: StartsWith 대신 Find 함수를 사용하여 검색어가 제목·요청자·요청번호 어디에 위치하더라도 검색되도록 수정하고, DelayOutput=false로 설정하여 실시간 검색이 가능합니다.
- 기간 필터 확장: 드롭다운 옵션에서 '사용자 지정'을 삭제하고 30일, 90일, 180일 선택지를 제공하도록 수정했습니다. 선택된 기간에 따라 varDateFrom이 자동 설정됩니다.
- 삭제/반려 처리 개선:
  - 의뢰자가 팀장 승인 대기 상태에서 요청을 삭제할 수 있는 기능을 추가하고, 삭제 전 확인 팝업을 도입했습니다.
  - 작업자가 InProgress 단계에서 반려할 수 있도록 버튼과 사유 입력창의 Visible 조건을 수정했습니다.
- 이메일 발송 보완: 작업 완료 시 의뢰자에게도 완료 알림 메일이 발송되도록 코드 수정. 대리 승인 시 작업자에게 메일이 발송되지 않던 문제를 Concurrent 블록으로 처리했습니다.
- 관리자 조회 권한 추가: 연구관리팀 팀장(seung.kim@tym.world)이 진행중과 완료 데이터를 모두 조회할 수 있도록 varIsViewer 변수를 추가하고, 갤러리 필터에 varIsWorker || varIsViewer 조건을 적용했습니다. 시작전 컬럼은 기존과 동일하게 본인 요청만 조회됩니다.
- 
### UI/UX 개선

- 색상 테마 재설계: 앱 배경과 패널·컬럼 헤더 색상을 새롭게 정의하여 기존 현황판과 차별화된 밝고 세련된 톤을 적용했습니다.
- 카드 템플릿 개선: 상태 배지를 상단에 배치하고 제목을 크게 표시하여 시선을 집중시키며, 요청자·요청일·마감일을 아래에 배치했습니다. 마감일 임박·지연 시 색상으로 표시합니다.
- 필터 패널 리디자인: 섹션 구분선 대신 왼쪽 4px 컬러바를 사용해 가독성을 높이고, 토글 라벨을 '완료 건 표시', '반려 건 표시'처럼 직관적인 문구로 변경했습니다.
- 역할별 상세화면 분리: 의뢰자, 팀장, 작업자 화면에서 하단 액션 카드(요청 처리/결재 처리/작업 처리)를 별도 컨테이너로 구현해 각 역할에 맞는 버튼만 보여주도록 했습니다.
